|                              |                                                                                                                            |
|:-----------------------------|:---------------------------------------------------------------------------------------------------------------------------|
| **Шаги для воспроизведения** | 1. Открыть Postman-коллекцию.<br>2. Найти эндпоинт `DELETE /api/2/item/:id`.<br/> 3. Изучить тело ответа при `404` ошибке. |
| **Фактический результат**    | В полях `result`, `status` написаны заглушки                                                                               |

## High (Высокий приоритет)

### 4) `DELETE /api/2/item/:id` возвращает `404` с телом ошибки `500` при попытке удаления несуществующего объявления
|                              |                                                                                                                                                     |
|:-----------------------------|:----------------------------------------------------------------------------------------------------------------------------------------------------|
| **Шаги для воспроизведения** | 1. Отправить `DELETE` запрос на `/api/2/item/{валидный_несуществующий_uuid}`. <br>2. Сравнить HTTP статус-код ответа и поле `status` в теле ответа.

| **Ожидаемый результат**      | Сервер должен возвращать статус `404 Not Found` с телом ощибки `404`.                                                                               |

## Critical (Критический приоритет)

### 12) `GET /api/1/item/:id` возвращает массив `[]` вместо одного объекта `{}`

|                              |                                                                                                                                                            |
|:-----------------------------|:-----------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Шаги для воспроизведения** | 1. Создать объявление через `POST /api/1/item` и получить его `id`.<br>2. Отправить `GET` запрос на эндпоинт `/api/1/item/{id}` с полученным `{id}`      . |
| **Фактический результат**    | Сервер возвращает статус `200 OK` и тело ответа в виде массива, содержащего один объект: `[ { "id": "...", ... } ]`.                                       |
| **Ожидаемый результат**      | Сервер должен возвращать статус `200 OK` и тело ответа в виде **одного объекта**: `{ "id": "...", ... }`.                                                  |
